# NEXT_TASKS.md — 釣果マップ 優先タスク一覧

> 更新日: 2026-05-03
> 方針: 個人用 → チーム共有 → 販売 の段階的展開を目標に、機能充実・情報精度向上・EC連動の三軸で進める。
> EC連動: 「場所×季節×魚種→ギア（ルアー・リール・ロッド）」の確率的分析をエージェントが自動実行し、地図上に表示。そのギアをECに誘導する。
> ギア収集: Webスクレイピング＋OCR（画像からの使用ギア情報読み取り）でエージェントが自動収集。

## フェーズ 1 — 情報精度の向上（エージェント改善）

### 収集精度

- [ ] **CollectorAgent にスクレイピングを追加**: DuckDuckGo スニペットだけでなく、anglers.jp の実ページを取得して本文を解析する（現在はスニペットのみ）
- [ ] **OCR対応の追加**: 釣果レポートの画像（SNS投稿・ブログ写真等）からギア情報をOCRで読み取る `OcrAgent` を新設（Claude Vision API活用）
- [ ] **ギア情報の収集項目追加**: 収集データに `lure_type`・`lure_name`・`rod`・`reel` フィールドを追加
- [ ] **釣り文脈情報の収集追加**: `time_of_day`（時間帯）・`depth_m`（棚の深さ）・`position`（岸釣り/ボート等）フィールドを追加
- [ ] **釣り人の準備・戦略情報の収集（わかる範囲で）**
  - `fishing_style`: 釣りスタイル（ルアー／フライ／エサ／テンカラ等）→ ブログ・レポートから抽出可能
  - `target_fish`: 狙っていた魚種（「〇〇を狙って」という記述から抽出）→ 抽出可能
  - `strategy`: 狙い・アプローチ（表層狙い・底狙い・流し釣り等）→ ブログ系なら抽出可能
  - `rod_count`: 竿の本数 → 画像OCRで読める場合のみ。不明時は省略
  - `tackle_count`: 持参ルアー・仕掛け数 → ほぼ不可能なため原則省略
  - `angler_level`: 投稿者の経験レベル推定（初心者／中級者／上級者）→ 文章の専門用語・タックル選択から推定。個人特定はしない
- [ ] **経験レベル別EC誘導**: レコメンドを経験レベルで分岐
  - 初心者 → 扱いやすいルアー・入門タックルを提案
  - 中級者以上 → テクニカルなアプローチ・専門ギアを提案
- [ ] **ユーザー自己申告フォーム**: アプリ登録時に「釣り歴・釣りスタイル」を任意入力できるようにし、レコメンド精度を向上（個人情報は収集しない設計）
- [ ] **信頼度スコアの付与**: 収集元（anglers.jp, ブログ, SNS等）ごとに信頼度を設定し、ValidatorAgent でフィルタリング
- [ ] **ソース URL の保存**: `catches_db.json` と `data.js` に `source_url` フィールドを追加して出典を記録・表示できるようにする

### 分析エージェント（新設）

- [ ] **AnalyzerAgent の新設**: 蓄積した釣果×ギアデータを集計し、スポットごとの確率分析を実行
  - 出力例: `{ spot: '〇〇ポイント', season: '春', top_fish: [{fish: 'ブラウントラウト', pct: 68}], top_lures: [{type: 'カエルルアー', pct: 52}, {type: 'ミノー', pct: 31}], best_time: '朝マズメ', avg_depth_m: 3.5 }`
- [ ] **季節定義の整備**: 春（3〜5月）／夏（6〜8月）／秋（9〜11月）／冬（12〜2月）でギア傾向を分類

### 検証精度

- [ ] **釣り場名の正規化強化**: ProcessorAgent のルックアップテーブルに洞爺湖・支笏湖の主要釣り場を網羅的に追加
- [ ] **AI検証レイヤーの追加**: ValidatorAgent に Claude によるデータ整合性チェック（「この魚はこの湖にいるか」「このルアーはこの季節に有効か」等）を追加

## フェーズ 4 — EC連動（マネタイズの核）

### ユーザーの2つのシナリオ

> **シナリオA（釣り前）**: マップで「どこで・何が・どのギアで釣れているか」を確認 → 釣り場とギアを決めて釣りに行く
> **シナリオB（釣れなかった後）**: 「なぜ釣れなかったか」をデータで振り返り → 次回のギア購入の参考にする（購買意欲が最も高いタイミング）

### 釣果 × 商品レコメンド

- [ ] **スポット分析パネル**: マーカークリックで「このスポットで釣れる確率・魚種・有効ギア・時間帯・棚の深さ」を表示
  - 例: `春の支笏湖〇〇ポイント｜ブラウントラウト 68%｜カエルルアー 52% / ミノー 31%｜朝マズメ｜棚3〜5m`
- [ ] **「釣れなかった人向け」レコメンド**: 閲覧中のスポット×季節で「よく使われているギア」をECリンク付きで表示
- [ ] **マーカーポップアップへのEC誘導**: 「このスポットで釣れているルアーを見る」ボタンを追加
- [ ] **季節・湖別おすすめ商品バナー**: サイドバーに現在の季節に合ったおすすめギアを表示

### EC基盤

> ⚠️ **収益化の具体的手法は別途検討**: 広告掲載 / Amazonアソシエイト等のアフィリエイトリンク / 動画コンテンツ連携 — いずれも選択肢として残し、フェーズ3以降に方針決定する。

- [ ] **収益化モデルの選定**: 広告・アフィリエイト・動画・自社EC等から方針を決定
- [ ] **商品マスタの整備**: 魚種・釣り方・季節と商品カテゴリのマッピングテーブルを作成（モデル決定後に着手）

### 精度と信頼性がECに直結する部分

- [ ] **「釣れた」情報の信頼性担保**: EC誘導の説得力は釣果情報の正確性に依存するため、自動収集データに信頼度スコアを表示
- [ ] **ユーザー投稿釣果へのタックル記録**: 手動入力フォームに「使用ルアー・タックル・時間帯・棚の深さ」フィールドを追加し、実績データとして蓄積

## 自動運用（常時更新）

> 情報は毎週自動更新。エージェントが人手なしで動き続けることが前提。

- [ ] **週次自動実行の設定**: Windows タスクスケジューラで毎週月曜 6:00 に `.\run.ps1 -Days 7` を自動実行

  ```powershell
  $action  = New-ScheduledTaskAction -Execute "PowerShell.exe" `
               -Argument "-NonInteractive -File `"$PWD\run.ps1`" -Days 7" `
               -WorkingDirectory $PWD
  $trigger = New-ScheduledTaskTrigger -Weekly -DaysOfWeek Monday -At "06:00"
  Register-ScheduledTask -TaskName "FishingMapUpdate" -Action $action -Trigger $trigger -RunLevel Highest
  ```

- [ ] **自動実行後の data.js 反映**: エージェント実行後に地図データが自動更新されることを確認
- [ ] **実行ログの自動ローテーション**: `logs/` が肥大化しないよう古いログを自動削除（30日以上前を削除）
- [ ] **死活監視**: 1週間エージェントが動いていない場合にアラートを出す仕組み
- [ ] **将来: クラウド移行**: Windows依存から脱却し、GitHub Actions や AWS Lambda で定期実行できるようにする（共有・販売フェーズで必須）
